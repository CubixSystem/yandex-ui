## Структура шапки

Если пройтись по разным сервисам Яндекса, то можно столкнуться с большим отличием между интерфейсами, однако структура шапки во многих сервисах схожа.
Схематично её можно представить вот так:
<img src="https://jing.yandex-team.ru/files/axaxaman/structure-header.png" />

- контент до шапки (баннер, алерт)
- логотип
- ссылки внутри сервиса
- поиск
- дополнительные ссылки
- блок с действиями

На эту структуру можно разложить большинство шапок Яндекса:

<img src="https://jing.yandex-team.ru/files/axaxaman/structure-example.png" />

Некоторые блоки могут отсутствовать, внутри может меняться расположение компонентов, поэтому базово шапка выглядит вот так:
<img src="https://jing.yandex-team.ru/files/axaxaman/anatomy-header.png" />

- Логотип
- Какой-то контент (навигация, поиск)
- Блок с действиями

React компонент реализует именно эту структуру и поставляет с собой ряд вспомогательных компонентов, чтобы можно было легко сделать шапку для сервиса:

- Поисковая форма (стрелочный вид и простой) (с отдельным view для Button и Textinput)
- Логотипы
- Навигационные табы
- Вспомогательные иконки
